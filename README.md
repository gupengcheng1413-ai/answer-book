# 答案之书 · 5.6寸长条屏 Demo

依据 Figma 设计稿像素级还原的「答案之书」前端 demo，目标载体为 5.6 寸长条副屏（**1640 × 348**），含完整的状态流转与原创动效。

## 两个版本

| 版本 | 过场动效 | 在线预览 |
|---|---|---|
| **v1** | 沙化吹散（原文案被风沙吹走 → 答案浮现） | https://gupengcheng1413-ai.github.io/answer-book/ |
| **v2** | 连翻 3 页书（仪式感更扣"答案之书"主题） | https://gupengcheng1413-ai.github.io/answer-book/v2/ |

v2 共享 `assets/` 资源目录；"再问一次" 改为返回 question 页让用户重新走完整流程。

## 启动

任选其一：

```bash
# 推荐：起本地静态服务（避免某些字体/资源跨域问题）
cd /home/pcgu3/answer-book && python3 -m http.server 8080
# 然后浏览器打开  http://localhost:8080/        ← v1
# 或者          http://localhost:8080/v2/     ← v2

# 或者直接双击 index.html
xdg-open index.html
```

> 打开后页面会自动 `scale` 适配窗口；想在原生 1:1 像素下查看，浏览器宽度 ≥ 1640，高度 ≥ 348 即可。

## 4 个状态

| 状态 | 描述 |
|---|---|
| **menu** | 入口菜单：1 张橙色「答案之书」实卡 + 3 张灰占位 |
| **question** | 提问页：左侧叠放书 + 标题 + 3 条疑问示例循环；右侧"心中默念，你思考的问题"+ 寻找答案按钮 |
| **answer** | 答案页：书翻开 + 机器人探头 + 大字答案 + 再问一次按钮 + 右上 X |
| **depleted** | 10 次用完：按钮置灰 +「今天的魔法已用完啦／明天再来和宇宙对话吧」 |

> `loading` 是 question→answer 的过场效果（非独立屏），约 1.6s，含按钮塌缩→宇宙力量来袭→书页 3D 翻动→星轨汇聚→答案浮现。

## 键盘调试

| 键 | 动作 |
|---|---|
| `1` | 跳转到入口菜单 |
| `2` | 跳转到提问页 |
| `3` | 跳转到答案页（随机抽一条） |
| `4` | 跳转到次数耗尽态 |
| `R` | 重置（清零 askCount） |

## 动效一览（核心"酷炫"部分）

- 星云背景双层错位漂移 + 椭圆光晕呼吸
- 入口卡片入场：blur 8→0 + Y+40→0 + scale .92→1，stagger
- 卡片 hover：辐射光斑 + 微浮起
- 提问页：多本书各自独立漂浮 / 标题 + 副星 + 圆点呼吸 / 3 条疑问交替循环
- 寻找答案按钮：conic 渐变光环旋转 + 涟漪扩散
- **S2→S3 过场**：橙光核扩散 → 横向飞向左侧书堆 → 书页 3D 翻动 → 12 道星轨向中心爆发
- 答案出现：机器人 spring 探头 → 答案逐字浮现（blur+translate） → 副标题延迟
- 关闭按钮 hover：90° 旋转
- 全部动效遵循 `prefers-reduced-motion`
